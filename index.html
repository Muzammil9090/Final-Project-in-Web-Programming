The CI/CD Methodology.

01. Source & Versioning

Code is pushed to GitHub where a Webhook triggers the Jenkins master. Version control ensures every change is traceable.

02. Automated Audit

SonarQube performs a deep static analysis to detect bugs, vulnerabilities, and code smells before build.

03. Containerization

Docker builds an immutable image from the source code, packaging all dependencies into a portable unit.

04. Remote Deployment

Jenkins uses SSH to securely connect to the production server, stops old containers, and spins up the new version with zero downtime.
